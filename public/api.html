API Routes

  • /api/create-user - Creates a user in the database. Going to be used in the Sign-Up section.
    • Method - POST
    • Body - { name, email, password }
    • Returns
      • Status 200, If the credentials were correct
      • Status 401, Data { error } - If the credentials were incorrect with error message

  • /api/login-user - Checks if the login credentials are correct. Going to be used in Login section
    • Method - POST
    • Body - { email, password }
    • Returns
      • Status 200, Data { userID, name, email, password, avatar } - If the credentials were correct. Data to be passed as props to <App />
      • Status 401, Data { error } - If the credentials were incorrect with error message

  • /api/create-post - Creates a new post
    • Method - POST
    • Body - { userID, latitude, longitude, title, content, tag, isAnonymous }

  • /api/get-posts?limit=6&page=9 - Gets all the posts for a location
    • Method - POST
    • Body - { latitude, longitude, tag }
    • Returns - Status 200, Data Array of { postID , author, avatar, title, content, tag, isAnonymous, comments}
    • comments - Array of { author, content, avatar } // Here userID means the user who commented on the the respective post

  • /api/get-posts-length - Gets all the posts for a location
    • Method - POST
    • Body - { latitude, longitude, tag }
    • Returns - Status 200, Data { length } where length is the number of posts next to the coordinates and with the filter

  • /api/create-comment - Creates a new comment on a post
    • Method - POST
    • Body - { userID, postID, content }
